POST

/schedule/upload

Feature 1

Upload an Excel (.xlsx) or CSV (.csv) file containing your shift schedule. Columns detected automatically by header name or position.

Required columns (header name or position):
  block_type / type / shift — night_shift, day_shift, evening_shift, off_day, transition_day
    Aliases: night, day, eve, off, transition
  start_time / start / from — ISO datetime e.g. 2026-03-22T19:00:00
  end_time / end / to — ISO datetime
Optional column:
  title / name / description — label for the shift
Sample CSV format: 
block_type,start_time,end_time,title
night_shift,2026-03-22T19:00:00,2026-03-23T07:00:00,ICU Night
day_shift,2026-03-23T09:00:00,2026-03-23T17:00:00,Day Follow-up
night_shift,2026-03-24T19:00:00,2026-03-25T07:00:00,ICU Night 2
POST

/schedule/upload-ical

Feature 2

Upload a .ics / .ical calendar file exported from Google Calendar, Apple Calendar, Outlook, NurseGrid, AMiON, HotSchedules, or Kronos. Each event becomes one schedule block.

Block type is inferred from event title keywords:
  night → night_shift
  day / morning / am shift → day_shift
  eve / afternoon / pm shift → evening_shift
  off / rest / leave / holiday → off_day
  transition / orient / train → transition_day
  Unrecognized → defaults to day_shift (warning added)
How to export from common systems:
  • Google Calendar → Settings → Import & Export → Export
  • Apple Calendar → File → Export → Export…
  • Outlook → File → Open & Export → Import/Export → Export to file → iCalendar
  • NurseGrid / AMiON → Schedule → Export / Sync → iCal link → download
GET

/schedule/google/auth → /callback

Feature 3

Two-step OAuth2 flow. Step 1 gets an auth URL. Step 2 (automatic via browser redirect) exchanges the code, fetches your Google Calendar events for the next N days, and imports them as schedule blocks.

Setup required (one-time):
1. Go to console.cloud.google.com → APIs & Services → Credentials
2. Create an OAuth 2.0 Client ID (Web application type)
3. Add Authorized redirect URI: http://localhost:8000/schedule/google/callback
4. Set GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET in your .env
5. Enable the Google Calendar API in the Google Cloud project

Open this URL in your browser to grant Google Calendar access:

After granting access, Google will redirect to the callback URL on your local server and automatically import your schedule.

GET

/schedule/outlook/auth → /callback

Feature 4

Two-step Microsoft OAuth2 flow using MSAL + Microsoft Graph API. Works with personal Outlook.com accounts and work/school Microsoft 365 accounts.

Setup required (one-time):
1. Go to portal.azure.com → App registrations → New registration
2. Name: Noxturn, Supported accounts: Personal + work/school
3. Redirect URI: Web → http://localhost:8000/schedule/outlook/callback
4. Certificates & secrets → New client secret → copy the Value
5. API permissions → Add → Microsoft Graph → Delegated → Calendars.Read
6. Set MICROSOFT_CLIENT_ID and MICROSOFT_CLIENT_SECRET in .env

Open this URL in your browser to sign in with Microsoft:

After signing in, Microsoft will redirect to the callback URL and automatically import your Outlook calendar events.

GET

/dashboard/today

Returns plan_mode, next_best_action, anchor_tasks, recovery_rhythm_label. Reads from in-memory state (generate a plan first).

POST

/tasks/event

Mark a task as completed, skipped, or expired. Copy a task_id from the /plans/generate response.

POST

/plans/replan

Regenerates next 1-3 actions after an anchor task is missed. Provide the same blocks used in generate.

POST

/wearables/import

Imports sleep metrics and computes recovery score. Upserts on date — safe to call multiple times.

GET

/personas/list & /personas/{id}

Browse all available worker personas. Persona context is injected into Claude to tailor task descriptions and priorities to the worker's role.

GET

/rag/retrieve

Token-overlap search over intervention cards and evidence chunks. No Supabase required.
